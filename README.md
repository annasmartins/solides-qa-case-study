# Solides - QA Case Study

Este repositório apresenta um estudo de caso de QA baseado na análise prática de usabilidade e comportamento funcional durante o preenchimento de cadastro em uma plataforma de recrutamento.

O objetivo deste projeto é documentar inconsistências identificadas durante a experiência de uso, com foco em gestão de sessão, navegação autenticada e clareza da interface.

## Objetivo
Registrar problemas observados no fluxo de cadastro de forma estruturada, aplicando boas práticas de QA manual e documentação de bugs.

## Escopo analisado
- Preenchimento de dados cadastrais
- Navegação entre páginas autenticadas
- Cadastro de habilidades

## Principais problemas encontrados
1. Expiração rápida de sessão com perda de dados preenchidos
2. Exibição momentânea da tela de login para usuário já autenticado
3. Falta de clareza no cadastro de habilidades e exibição confusa do nível

## Competências demonstradas
- Testes manuais
- Identificação de bugs
- Escrita de bug reports
- Análise de usabilidade
- Pensamento crítico
- Foco na experiência do usuário

## Observação
Este material foi elaborado como estudo de caso educacional e profissional, com base em experiência real de uso da plataforma, sem exposição de dados pessoais ou informações sensíveis.
